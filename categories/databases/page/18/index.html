Sanity-check features in MySQL

Fri, Dec 25, 2009 MySQL has a couple of sanity-check features to help keep you from doing dumb things. max_join_size is a configuration option for the mysqld server program. It throws an error if you write a query that the optimizer estimates will examine more than this number of rows. –safe-updates is a command-line option to the mysql client program. It throws an error if you write an UPDATE or DELETE without a) a WHERE clause that refers to an indexed column or b) a LIMIT clause.
